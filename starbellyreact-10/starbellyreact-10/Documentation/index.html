Local Development Environment Setup

In order to kickstart the development of your web project with Starbelly, the very first thing is to setup an React Next.JS development environment.

System Requirements:

  • Node.js 16.14 or later.
  • macOS, Windows (including WSL), and Linux are supported.

Installation guide: read here

Getting Started

Name Description
next.config.js Used for configuration of Next.js project specific options. You can check all the available configuration options here.
jsconfig.json Used for configuration absolute imports and module path aliases.
node_modules All external modules used are here. Do not mess with this folder, as it is auto-generated by using npm install.
package.json Contains all dependencies used for production and development.
public Contains all Static files for project, assets (images, fonts, video, favicon) and 3rd party styles(css) files.
src Contains all React Jsx code, assets and basically everything the end user will have access to.
  • Node.js and NPM : If NodeJS is not installed then download it from https://nodejs.org/. NPM comes bundled with Node.js

NPM Packages

  • Before proceding you'll need to install npm packages. You can do this by running npm install from the root of your project to install all the necessary dependencies.

Development Server

  • Run npm start for a dev server. Navigate to localhost:3000/. The src will autometically reload if you change any of the source files.

Building

  • Run npm run build to build the project. The build artifacts will be stored in the .next/ directory. Use the run build flag for a production build.

More Help

  • To get more help on the react use npm --help or go check out the React Support.

Export to static HTML

  • If you need export HTML static files from your Next.js project just go to /package.json file and replace "build": "next build", line to "build": "next build && next export", and run npm run build command again, readmore details here.

Data Info

All site data (content, text constants, links, settings, images paths and etc) can be found in the "/src/data/" directory:

  • Global Settings (/src/data/app.json)

    Contains default site settings, header and footer data, social links and contacts constants

  • Posts data (/src/data/posts/*md)

    Contains markdown based files of posts for Blog Type Content

  • FAQ data (/src/data/faq.json)

    Contains json based files of FAQ Type Content

  • Gallery data (/src/data/gallery.json)

    Contains json based files of Gallery Type Content

  • Menu data (/src/data/menu.json)

    Contains json based files of Menu Type Content

  • Products data (/src/data/products.json)

    Contains json based files of Product Type Content

  • Reviews data (/src/data/reviews.json)

    Contains json based files of Review Type Content

  • Sections data (/src/data/sections/*.json)
    Sliders data (/src/data/sliders/*.json)

    Contains sections content data (text constants, image paths and etc) for available components:
    - sections
    - sliders

Fonts Used

By default, the template loads Rubik font from Google Web Font Services, you can change the font with the one that suits you best.

Font code can be found in the "/src/app/layout.jsx" file:


import { Rubik, Monoton } from 'next/font/google'

const rubik = Rubik({
    weight: ['300', '400', '500', '600', '700', '800', '900'],
    style: ['normal', 'italic'],
    subsets: ['latin'],
    variable: '--font-rubik',
    display: 'swap',
})

const monoton = Monoton({
    weight: ['400'],
    subsets: ['latin'],
    variable: '--font-monoton',
    display: 'swap',
})
